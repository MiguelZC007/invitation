# i18n Rules (next-intl 4.x)

## Configuración
- Locales: `es` (default), `en`.
- Prefix: `always` — todas las rutas tienen locale (`/es/...`, `/en/...`).
- Configuración en `src/i18n/routing.ts` (defineRouting) y `src/i18n/request.ts` (getRequestConfig).

## Uso en componentes
- Client components: `useTranslations("namespace")` — retorna `t("key", { ...values })`.
- Server components: `getTranslations("namespace")` desde `next-intl/server`.
- Namespaces corresponden a keys de primer nivel en los JSON de mensajes.
- En pages/layouts async: `setRequestLocale(params.locale)` al inicio.

## ICU Message Format
- Fechas: `{orderDate, date, short}` o `{orderDate, date, ::yyyyMMMd}` en el JSON.
- Números: siempre `{count, number}` para localización — evitar `"{count} followers"` sin formatear.
- Pluralización: `{count, plural, =0 {...} =1 {...} other {...}}`.

## Archivos de mensajes
- `src/messages/es.json` y `src/messages/en.json`.
- Mantener ambos archivos sincronizados (mismas keys).
- Estructura plana o un nivel de anidación máximo.
- Keys en inglés (camelCase): `"welcomeMessage"`, no `"mensaje_bienvenida"`.

## Rutas
- Todas las páginas bajo `src/app/[locale]/`.
- Validar locale con `hasLocale(routing.locales, locale)` en layouts.
- `setRequestLocale(locale)` obligatorio para static rendering.
- `generateStaticParams()` genera rutas para todos los locales.
